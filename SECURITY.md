# Politica de seguridad

## Reporte de vulnerabilidades

Si encontraste una vulnerabilidad en un producto 0E3, no abras un issue publico con detalles explotables. Contactanos por email:

- ceroes3group@gmail.com

Inclui:

- Producto o repositorio afectado.
- Version, rama o commit si aplica.
- Descripcion del impacto.
- Pasos de reproduccion.
- Evidencia tecnica minima necesaria.

## Secretos y credenciales

Nunca subir:

- Archivos `.env` reales.
- Claves API.
- Service accounts.
- Keystores Android.
- Certificados privados.
- Tokens de GitHub, Firebase, Mercado Pago, Google Cloud u otros proveedores.

Usar siempre `.env.example` para documentar variables sin valores reales.

## Estado

La politica formal de seguridad y tiempos de respuesta queda pendiente hasta la apertura publica de los repositorios.
