# Contribuir a 0E3

Gracias por colaborar con 0E3.

## Flujo recomendado

1. Crear una rama desde `main`.
2. Hacer cambios pequenos y revisables.
3. Ejecutar analisis, tests o build local segun el stack.
4. Abrir pull request con descripcion clara.
5. Esperar revision antes de mergear.

## Convenciones

- No subir secretos ni credenciales.
- No mezclar cambios de producto con cambios de infraestructura salvo que sea necesario.
- Documentar decisiones relevantes en `docs/`.
- Mantener README y `.env.example` actualizados.

## Commits

Usar mensajes claros y orientados al motivo del cambio. Ejemplos:

- `Add base landing structure`
- `Fix POS session timeout handling`
- `Update brand asset guidelines`

## Licencia

La licencia queda pendiente de definicion institucional. No asumir licencia open source hasta que exista un archivo `LICENSE`.
